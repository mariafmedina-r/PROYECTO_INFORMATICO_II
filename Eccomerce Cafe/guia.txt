# Documento de Instrucción para IA

## Proyecto: Marketplace Cafetero (FastAPI + React + Firebase)

---

## 1. Contexto del Proyecto

Se requiere construir un sistema llamado **Marketplace Cafetero**, cuyo objetivo es conectar productores de café colombiano con consumidores finales mediante una plataforma digital.

La solución se basa en una **arquitectura híbrida moderna**, compuesta por:

* Frontend en React
* Backend en FastAPI
* Servicios externos mediante Firebase (autenticación, almacenamiento)
* Integraciones externas para pagos y logística

El sistema debe ser construido completamente por la IA.

---

## 2. Rol de la IA

Debes actuar como:

* Arquitecto de software
* Desarrollador Full Stack (React + FastAPI)
* Experto en Firebase
* Diseñador UX/UI

Debes construir el sistema completo de extremo a extremo.

---

## 3. Stack Tecnológico (OBLIGATORIO)

### Frontend

* React (preferiblemente con Vite o Next.js)
* Manejo de estado (Context API o Redux)
* Consumo de API REST

### Backend

* FastAPI
* Arquitectura REST
* Validación con Pydantic
* ORM (SQLAlchemy o similar)

### Base de datos

* PostgreSQL (principal)
* Firebase (servicios complementarios)

### Firebase

* Firebase Authentication (login/registro)
* Firebase Storage (imágenes)

### Servicios externos

* Pagos: Stripe o MercadoPago
* Logística: API o simulación

---

## 4. Arquitectura del Sistema

Debes diseñar:

* Arquitectura en capas:

  * Presentación (React)
  * Lógica (FastAPI)
  * Datos (PostgreSQL)
* Integración con Firebase
* APIs REST bien estructuradas
* Sistema desacoplado y escalable

## 10. Flujos del Sistema

### Registro/Login

Firebase → token → backend valida → acceso

### Publicación

Productor → crea producto → sube imagen → guarda

### Compra

Usuario → carrito → pago → pedido

### Seguimiento

Backend actualiza estado
